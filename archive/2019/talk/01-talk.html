Web Application Firewall in DevOps

Franziska Bühler

Web Application Firewalls (WAF) verursachen False Positives und Probleme in der Produktion. Und zu neuen agilen Vorgehen und DevOps passen sie schon gar nicht mehr. Leider wird das oft befürchtet. Zu Unrecht, wie ich meine. Wie wäre es, wenn wir mit der WAF den Shift Left Ansatz verfolgen würden und uns die produktive Einführung einer WAF keine Sorgen bereiten würde?
Wenn ich von WAF spreche, meine ich ModSecurity mit dem OWASP Core Rule Set. Für die Präsentation verwende ich ausserdem die extra verwundbar programmierte Web Applikation Pixi des OWASP DevSlop Projektes. Ich zeige auf, wie Pixi mit dem CRS davor in eine Continuous Integration (CI) Pipeline eingebunden und getestet werden kann. Der Entwickler oder die Entwicklerin erhalten bereits während der Testphase früh und oft Feedback über das Zusammenwirken ihrer Applikation mit dem CRS.
In diesem Vortrag lernen wir das OWASP ModSecurity Core Rule Set (https://coreruleset.org), das OWASP DevSlop Projekt (https://devslop.co), die Open Source Version von TestCafe (https://github.com/DevExpress/testcafe) und die Continuous Integration Platform CircleCI (https://circleci.com/) kennen.

Franziska arbeitet für die Schweizer Firma Puzzle ITC, wo sie ihre Leidenschaften für IT-Security, DevOps und Open Source Software verbinden kann. Sie hat viel Erfahrung im Bereich Web Application Firewall und Reverse Proxy unter anderem durch ihre Mitentwicklung des OWASP ModSecurity Core Rule Sets.
Sie ist ebenfalls Teil des OWASP DevSlop Teams, wo sie das CRS in Pixi's Continuous Integration Pipeline integriert hat. Dadurch erhalten die Entwickler die Gelegenheit, die Applikation und die WAF früh im SDLC aufeinander abzustimmen.
Ausserdem bloggt sie, spricht an Konferenzen und gibt technische Trainings. Wenn sie nicht gerade hochoptimierte Regexes disassembliert oder technische Bücher wälzt, verbringt sie sehr gerne Zeit mit ihrer Familie.
