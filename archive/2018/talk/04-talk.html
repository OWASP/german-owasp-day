Docker Threat Modelling und Top 10

Dirk Wetter

Docker Container sind hip. Nicht ohne Grund, bieten sie viele Vorteile für die Entwcklung. Nur manchmal wird mehr Dev als Ops gemacht -- Entwicklern sind Themen wie System- und Netzwerksecurity meistens nicht in die Wiege gelegt. Docker Security ist aber ein klassisches System- und Netzwerksecurity-Thema.

Auch wenn es viele bedrohliche Informationen da draußen gibt, was Docker anegblich alles für Gefahren mitbringt -- per se ist Docker eine Containment-Technologie, die diverse Sicherheitsvorteile und keine -Nachteile mit sich bringt. Es kommt nur drauf an, wie man mit Docker Containern umgeht und wie man die gröbsten kritischen Punkte umschifft.

Daher ist eine grundsätzliche Heransgehensweise gefragt.

Der Sprecher wird daher zunächst eine Bedrohungsanalyse vornehmen, um daraus die wichtigsten zehn Security Bullet Points abzuleiten. Angefangen von Do's und Dont's ehr auf Baseline-Niveau bis zu fortgeschtrittenen Punkten, die weit darüber hinausgehen.

Der Vortrag basiert auf einer Reihe praktischer Erfahrungen beim Auditieren von Docker-Umgebungen.
