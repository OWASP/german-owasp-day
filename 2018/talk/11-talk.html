PostScript Undead: Pwning the Web with a 35 Years Old Language

Jens Müller

PostScript ist eine Turing-vollständige Programmiersprache aus den frühen 80er Jahren. Ursprünglich für den Einsatz in Laserdruckern entwickelt, waren PostScript Dateien über lange Zeit das de-facto Standardformat für wissenschaftliche Publikationen. Mit dessen Ablösung durch PDF um die Jahrtausendwende geriet PostScript zwar weitgehend in Vergessenheit, war aber niemals wirklich tot: Der Baustein zur Ausführung von PostScript Code ist nach wie vor auf fast allen Unices vorhanden. Hier fristet er ein Schattendasein und wartet darauf, dass frischer Code sein volles Potenzial erweckt. Doch welche Webanwendung würde einen PostScript Interpreter mit beliebigem User-Input koppeln?

Viele Anwendungen tun dies -- meist ohne es zu wissen. Durch eine Verkettung fragwürdiger Designentscheidungen in Web-Frameworks und Bildverarbeitungsbibliotheken sowie durch wenig bekannte `PostScript Downgrade' Angriffe innerhalb von PDF Dateien kann die halbtote Sprache PostScript wieder zum Leben erweckt werden -- etwa um Bitcoins auf dem Webserver zu minen.

Im Vortrag wird gezeigt, dass beliebte Webseiten wie Wikipeda, Microsoft OneDrive und Google Mail durch bösartige PostScript File Uploads angegriffen werden können. Neben einer systematischen Sicherheitsanalyse legitimer Features der Programmiersprache wurde eine Untersuchung des verbreitetsten PostScript Interpreters -- Ghostscript -- durchgeführt. Die Angriffe reichen von DoS, über Information Disclosure und File Inclusion bis hin zu Remote Command Execution. Eine Analyse von 100 bildverarbeitenden Webseiten zeigt, dass 56% anfällig für mindestens einen Angriff sind. Hack like it's 1984!
